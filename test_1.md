## Тестовое задание №1

**Бизнес-проблематика:** 
Клиент продает квартиры. У него есть CRM (amoCRM), сайт с формой заявки и Telegram-канал/группа/бот для уведомлений менеджеров. Нужно связать это воедино.

**Задача:**
Создать микросервис на Symfony или FastAPI, который:
1. Принимает webhook от формы сайта (POST с JSON: имя, телефон, интересующий объект) 
2. Валидирует данные
3. Отправляет лид в [amoCRM](https://www.amocrm.ru/developers/content/crm_platform/leads-api#leads-add)
4. Отправляет уведомление в Telegram
5. Логирует все действия в БД

Если нужен тестовый аккаунт amoCRM, его можно зарегистрировать по ссылке [amocrm.ru](https://www.amocrm.ru/)

**Критерии приемки:**
- Развертывание через docker-compose
- Создание сделки в amoCRM (в .env должна быть переменная для long-token из amoCRM)
- Сообщение в телеграм-канал/группу/бот
- Наличие Readme.md с описанием работы

**Со звезочкой:**
- Добавить простую админку для просмотра логов
- Добавить форму из пункта 1 Задачи
- Реализовать контроль дублей для amoCRM (Поиск контакта перед созданием сделки. Если контакт есть, то обновлять уже существующую сделку)

**Оценка:** 2-4 часа работы
